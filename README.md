# Self-harness

A `PERSON.md` for myself. Same idea as `CLAUDE.md`, different audience.

## Why this exists

I've been working with AI agents for a while and noticed something weird: my throughput went up, but my sense of being in control went down.

Early in a project, things feel great. Halfway through, I start losing track of my own codebase. By the time I'm wrapping up, I'm tired in a way I can't really explain — I haven't done that much, but I've been half-watching the agent the whole time. Somewhere in there I started doubting whether I still know how to code.

Eventually I figured out it wasn't the agent. It was me — or more specifically, how I was working with it. I was treating every task with the same half-attention. Nothing got real focus, nothing got real handoff, and I was quietly outsourcing decisions I should've been making myself.

We write `CLAUDE.md` files because we know the model drifts. Wrong assumptions, no pushback, lost context, fake confidence. But people drift too. Same failure modes, honestly.

So I wrote one for myself. Called it `PERSON.md`.

## What's in here

[`PERSON.md`](./PERSON.md) — four rules:

1. Hand off the boring stuff
2. Stay close to the code that matters
3. Make your own calls
4. Don't use AI to numb yourself

Each one comes with a "slipping when." Catching it is the hard part — the signal has to be specific or I'll miss it.

## For the record

Not a tutorial. Not advice. Not a framework. **It's mine.** I'm putting it on GitHub because making it public is part of the harness — easier to ignore something nobody's seen.

If you're using AI agents and feel like you're getting faster but losing your grip, fork this and write your own `PERSON.md`. Don't copy mine. Your failure modes aren't my failure modes.
